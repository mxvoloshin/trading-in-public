I am starting **Trade in Public**: an automated swing-trading system built the way I would build any serious software system, with clear requirements, tests, logs, safety gates, review cycles, and honest postmortems.

This blog will track the route from idea to a small live automated account: learning Python, choosing a broker/API path, validating the workflow, running the first version from a Mac mini, and documenting what breaks along the way.

## Ground Rules

- This is not financial advice.
- I will start with small capital and treat risk controls as part of the product.
- Every strategy needs a written hypothesis before a backtest.
- Backtest results must include assumptions, not just screenshots of profit curves.
- Failures are part of the record.
- I will not publish credentials, sensitive account details, or real-time recommendations.

## What Comes Next

The first milestone is the boring but important foundation: pick the broker/API path, define what safe automation means, create the Python project skeleton, and start a weekly public review rhythm.
